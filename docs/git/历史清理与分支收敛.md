## Phase 4: 历史改写与远端同步（第一轮）

### 步骤 4.1 检查 filter-repo 工具可用性

```bash
git filter-repo --version
# 或
python3 -m git_filter_repo --version
```

### 步骤 4.2 从完整历史移除脏文件

```bash
git filter-repo --force \
  --path build --path install --path log \
  --path .vscode --path .cache --path tmp \
  --path .idea --path .colcon \
  --invert-paths
```

**说明**:
- `--invert-paths`: 保留所有路径**除了**指定的路径
- `--force`: 跳过安全检查（已有备份分支）
- 处理了 37 个提交

**输出**:

## Phase 5: 分支收敛为仅 master

### 步骤 5.1 删除所有本地非 master 分支

```bash
echo 'Local branches before:'
git for-each-ref refs/heads --format='%(refname:short)'

# 删除除 master 外的所有本地分支
for b in $(git for-each-ref refs/heads --format='%(refname:short)' | grep -v '^master$'); do
  git branch -D "$b"
done

echo 'Local branches after:'
git for-each-ref refs/heads --format='%(refname:short)'
```

### 步骤 5.2 删除 origin 远端的所有非 master 分支

```bash
echo 'Remote origin branches before:'
git ls-remote --heads origin

# 删除除 master 外的所有远端分支
for rb in $(git ls-remote --heads origin | awk '{print $2}' | sed 's#refs/heads/##' | grep -v '^master$'); do
  git push origin --delete "$rb"

echo 'Remote origin branches after:'
git ls-remote --heads origin
```

## 常见问题与解决方案

### Q1: 执行 filter-repo 后远端消失

**原因**: filter-repo 默认移除所有远端配置。

**解决方案**:
```bash
git remote add origin <remote-url>
```

### Q2: push 时收到 "stale info" 错误（--force-with-lease 失败）

**原因**: 本地远端跟踪信息过期或与实际远端历史不一致。

**解决方案**:
```bash
git fetch origin master
git push --force-with-lease=refs/heads/master origin master
```

或明确指定上一个已知的远端提交：
```bash
git push --force-if-includes origin master
```

### Q3: 历史改写后，upstream 无法同步

**原因**: 可能当前账号对 upstream 仓库无写权限。

**解决方案**: 联系有权限的账号或改用拥有权限的账号执行同样的 force-push 和分支删除步骤。

### Q4: 如何回滚某一次历史改写

**方法**: 使用之前创建的备份分支或标签。

```bash
# 查看备份分支/标签
git branch -a | grep backup
git tag | grep before

# 回滚到某个备份点
git reset --hard before-cleanup-20260329-203058
# 或
git checkout -b recovery-branch backup/pre-cleanup-20260329-203058
```
